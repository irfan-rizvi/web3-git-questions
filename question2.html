Question 2: Compare CLI vs. GUI clients for working with Git

Git is a popular DVCS and it could be controlled with a bash CLI or some GUI interfaces.

  • Git CLI is very powerful and robust. The best thing is that it does not have many commands to remember. Only a few are used frequesntly to do most of the work.
  • There are many GUI interfaces to be used with the Git like gitk, Git GUI etc
  • gitk is more like git log --graph and shows history of commits etc. Git GUI on the other hand is a GUI interface to craft commits.
  • One disadvantage of using GUI interfaces for Git is that you do not understand the Git properly. As a programmer we should be comfortable with challenges and using a GUI is not as challenging as using Git through CLI. Using Git through CLI gives more controll as well.
